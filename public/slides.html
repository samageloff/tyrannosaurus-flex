What is Flexbox?

It's a "new" layout module in CSS3.

New = 8 years old.

8 years old = you can now use it.

We use it here at Ambassador.

Does flexbox make things better?

  • Yes.
  • It helps to position items in horizontal or vertical stacks in the browser.
  • It determines the most efficient way to lay out, align and distribute space among items in a container.
    • (even when their size is unknown or dynamic).
  • Notoriously difficult to do in a browser for many years
  • tables > floats > flex
  • W3C started the working draft in June 2009
  • Has become widely adopted as a web standard

Why should we use it?

  • Designers and developers find it easy to use.
  • Less HTML and CSS to achieve complex layouts.
  • Handle many layout scenarios with one codebase.
  • It's direction agnostic.
  • A little bit to get used to
  • Layout scenarios: could be device size, or could be toggling between states on a desktop app.
  • Table grid/columns, sorting
  • https://philipwalton.github.io/solved-by-flexbox/

The pretty basic example.

Create a "flex-container":


							

						

							.flex-container {
							  display: flex;
							}

Add a few "flex items":


							

							  
1

							  
2

							  
3

							

						

							.flex-container {
							  display: flex;
							}

							.flex-item {
							   flex: 1;
							}
  • Some of you may be familiar with this "display" property
  • Traditionally: block (vertical), inline (horizontal)
  • The flex container gives you access to properties that can handle either scenario.
  • Adding `display: flex` to an element provides access to the "Flex API" for it's child elements.
  • `flex: 1` is the flex "grow" property. Tells each item to fill the space equally amongst themselves.
  • Adding items will tell the browser to equally distrubute space no matter how many.
  • Makes the flex item flexible and sets the flex basis to zero, resulting in an item that receives the specified proportion of the free space in the flex container. If all items in the flex container use this pattern, their sizes will be proportional to the specified flex factor.

THe Flex "API".

Flex container

align-items: [baseline | center | flex-end | flex-start | stretch]

flex-direction: [row | column | row-reverse | column-reverse]

flex-wrap: [wrap | wrap-reverse]

justify-content: [center | flex-start | flex-end | space-around | space-between]
